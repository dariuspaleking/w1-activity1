A starter page with a working header, navigation, and About section. You'll fill in the missing sections and styles so the page reflects your own choices, not the placeholder.


Or double-click `index.html` from your file explorer.

## Verify

Your activity is done when, in the browser:

- [ ] An About paragraph clearly explains the website vs. webpage difference in your own words.
- [ ] Three concept cards (HTML, CSS, JavaScript) sit side-by-side in the grid.
- [ ] Clicking the "Practice" nav link scrolls to a real section with a heading, list of 3 items, and one external link.
- [ ] Each section appears as a card with a subtle shadow on the purple background.
- [ ] Concept cards use a purple gradient with white text and lift slightly when you hover.
- [ ] The page is readable and balanced when you resize the window to phone width.

## Stretch

Pick one and note it in your reflection:
- Add a fourth concept card for a tool you use (Git, VS Code, Figma).
- Swap the purple theme to a palette you choose; explain the colors.
- Add a smooth-scroll behavior so nav links glide instead of jump.

## 🪞 Reflect on Your Work

Answer in 2-3 sentences each, in this README under your TODO commits. Your tutor reads these as part of grading.

1. **What did you learn that you didn't know before?** Name the most surprising thing — a CSS property that behaved unexpectedly, an HTML structure rule, a hover detail.
2. **How did you collaborate with AI?** If you used Claude / ChatGPT / Cursor / Copilot, what part of the work did *you* contribute — the prompt, the verification, the design decision, the bug-fix? If you didn't use AI, what was the hardest thing to figure out alone?
3. **How do you know your code works?** Describe one specific check you ran — a resize you tested, a hover you watched, a screenshot you took.

> AI is a great collaborator. Owning your thinking, verifying the output, and explaining your design choices is what *learning* looks like in this course.
